# killercoda-scenarios

Hands-on labs for Killercoda. Scenarios use English instructional text (see [AGENTS.md](AGENTS.md)).

## Suggested learning path (Kubernetes)

1. **[kubernetes-1node](kubernetes-1node/)** — Beginner: list pods cluster-wide on a single-node kubeadm environment.
2. **[kubernetes-2node-multi-step-verification](kubernetes-2node-multi-step-verification/)** — Intermediate: create and delete a pod on a two-node cluster with step checks.
3. **[kubernetes-volumes](kubernetes-volumes/)** — Intermediate: PVCs, pods, and file CRUD on mounted storage.
4. **[kubernetes-dashboard](kubernetes-dashboard/)** — Intermediate: deploy the web UI with lab-only security settings.

Other content:

- **[linux-fundamentals](linux-fundamentals/)** — Beginner: Linux basics (shell, files, users, permissions, packages).

## Scenario index

- [kubernetes-1node](kubernetes-1node/) — Skill level: Beginner — List workloads with kubectl on a single-node kubeadm cluster.
- [kubernetes-2node-multi-step-verification](kubernetes-2node-multi-step-verification/) — Skill level: Intermediate — Pod lifecycle on a two-node cluster with verification scripts.
- [kubernetes-volumes](kubernetes-volumes/) — Skill level: Intermediate — PersistentVolumeClaims and file operations.
- [kubernetes-dashboard](kubernetes-dashboard/) — Skill level: Intermediate — Deploy and access the Dashboard (lab-oriented settings).

## About Kubernauten

Kubernauten is a community of Kubernetes and cloud-native enthusiasts.
We learn together, we grow together.

These Killercoda scenarios exist to provide practical, hands-on labs for
learning Kubernetes and Linux concepts in a safe, browser-based environment.
They are used in workshops, meetups, and for self-paced learning.
